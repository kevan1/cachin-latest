## Required User Education

Explain this in both places:

- **Website / landing page**: before signup, explain how paying works, when KYC is required, what happens if a payment fails, and how support/refunds work.
- **App onboarding and payment flow**: teach only the minimum needed before the first payment, then show clear pre-payment confirmation, receipt, and support CTA.

## Minimum Trust Copy

Use:

> Every payment has a clear receipt and direct support. If something goes wrong, Cachin support is available through Crisp.

Avoid:

> Payments can never fail.

Avoid:

> Instant refunds guaranteed.

Say that only when the operational process and provider obligations are documented.
